## Bug Details

### BUG-1: LHDN Dev Tools Page Blank (HIGH)

**Page:** `/app/lhdn-dev-tools`
**Screenshot:** `lhdn-dev-tools-full.png`
**Expected:** 6 interactive panels (System Status, Connection Test, Manual Triggers, Exemption Tester, Recent Submissions, Re-submit)
**Actual:** Only sidebar with "LHDN Payroll" renders. Main content area is completely empty `<div id="body"></div>`.
**Root cause:** `frappe.pages['lhdn-dev-tools']` is NOT FOUND. The `getpage` API returns 200 but the page script is empty (0 bytes). This is because the page JS file is at the wrong path in the container due to triple-nesting (BUG-4).
**Regression test:** REG-14 (currently failing as expected)
**Fix:** Rebuild Docker image with correct COPY path, or manually symlink the page directory.

### BUG-2: Admin Name "Hacked" (FIXED)

**Where:** Sidebar user menu
**Cause:** SEC-06 CSRF test's `tearDown()` didn't restore `first_name`. Run during previous test execution.
**Fix applied:** `bench execute frappe.client.set_value "User" "Administrator" "first_name" "Administrator"`
**Regression test:** REG-13 (passing)

### BUG-3: Sidebar JS Error (HIGH)

**Error:** `TypeError: Cannot read properties of undefined (reading 'public')` at `sidebar_item.js:36`
**Stack:** `get_path → make → constructor → make_sidebar_item → add_item → create_sidebar`
**Cause:** A workspace sidebar item references undefined config. Likely the LHDN Payroll workspace's shortcuts have broken link definitions (workspace is missing from DB).
**Impact:** Error is caught by Frappe — doesn't crash the page, but sidebar items for LHDN module don't render.

### BUG-4: Triple-Nested Directory (HIGH)

**Container path (WRONG):**
```
/apps/lhdn_payroll_integration/
  lhdn_payroll_integration/        ← pyproject.toml, README
    lhdn_payroll_integration/      ← hooks.py, page/, doctype/ (SHOULD be here)
      lhdn_payroll_integration/    ← another copy
```

**Expected:**
```
/apps/lhdn_payroll_integration/
  lhdn_payroll_integration/        ← hooks.py, page/, doctype/
```

**Impact:** Frappe finds `hooks.py` at level 3, but looks for `page/` at level 2. Page files, templates, and reports aren't found.
**Fix:** Correct the `COPY` directive in `Dockerfile.myinvois` or restructure the local repo directory.

### BUG-5: LHDN Payroll Workspace "Not Found" (HIGH)
