## Codebase Patterns

- `bench new-app` is interactive: needs App Title, Description, Publisher, Email, License, GitHub Workflow (N), Branch Name
- Use `printf '...\n' | bench new-app ...` with all answers including branch name and workflow=N
- Must enable testing: `bench --site frontend set-config allow_tests true` before running tests
- `bench new-app` creates modules.txt with default module name automatically
- Test file goes in `apps/<app>/<app>/tests/test_*.py`
- FrappeTestCase import: `from frappe.tests.utils import FrappeTestCase`
